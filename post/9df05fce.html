Angular CLI 使用教程指南参考

发表于 2018-05-17 |

Angular CLI 现在虽然可以正常使用但仍然处于测试阶段. Angular CLI 依赖 Node 4 和 NPM 3 或更高版本.

安装

要安装Angular CLI你需要先安装node和npm，然后运行以下命令来安装最新的Angular CLI：

注意：Angular CLI 需要Node 4.X 和 NPM 3.X 以上的版本支持。

## 基本用法

你可以通过 Angular CLI 的 help 命令来获取相关的命令信息.

## ng get

<table>
<thead>
<tr><th>命令</th><th>描述</th></tr>
</thead>
<tbody>
<tr>
<td>`ng get <path1, path2, ...pathN> [options]`</td>
<td>从Angular CLI配置获取值</td>
</tr>
</tbody>
</table>

pathN是一个有效的JavaScript参数路径，例如“users[1].userName”。 如果未设置该值，将显示“undefined”。 此命令默认情况下仅在项目目录中工作。

<table>
<thead>
<tr><th>参数</th><th>描述</th></tr>
</thead>
<tbody>
<tr>
<td>`--global`</td>
<td>返回全局配置值，而不是本地配置值（如果都设置）. 此选项还可以使命令在项目目录外工作</td>
</tr>
</tbody>
</table>

## ng set

<table>
<thead>
<tr><th>命令</th><th>描述</th></tr>
</thead>
<tbody>
<tr>
<td>`ng get <path1=value1, path2=value2, ...pathN=valueN> [options]`</td>
<td>在Angular CLI配置中设置值</td>
</tr>
</tbody>
</table>

默认情况下，如果在项目内部运行，则设置项目配置中的值，如果不在项目内部，则失败。 pathN参数是一个有效的JavaScript路径，如“users [1] .userName”。 该值将被强制转换为正确的类型，或者如果类型无法强制，则会抛出错误。

<table>
<thead>
<tr><th>参数</th><th>描述</th></tr>
</thead>
<tbody>
<tr>
<td>`--global`</td>
<td>设置全局配置值，而不是本地配置值。 这也使`ng set`可以在项目之外工作。</td>
</tr>
</tbody>
</table>

## ng build

构建工件将存储在/dist目录中。

ng build可以指定构建目标（–target = production或–target = development）和要与该构建一起使用的环境文件（–environment = dev或–environment = prod）。 默认情况下，使用开发构建目标和环境。

<pre class="hljs sql"><code># 这是生产构建
ng build <span class="hljs-comment">--target=production --environment=prod
ng build <span class="hljs-comment">--prod --env=prod
ng build <span class="hljs-comment">--prod

# 这是开发构建
ng build <span class="hljs-comment">--target=development --environment=dev
ng build <span class="hljs-comment">--dev --e=dev
ng build <span class="hljs-comment">--dev
ng build</span></span></span></span></span></span></code>```
